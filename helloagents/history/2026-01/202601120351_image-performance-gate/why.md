# 变更提案: 图片性能规范与原子设计文档补齐

## 需求背景

本项目以“纯静态 / 无框架 / Local-first / PWA 离线”为核心约束，追求稳定交付与高性能。

在现有架构下，仍存在两类可预防的常见问题：

1. **图片加载策略缺省导致的性能波动**
   - 现状：静态 HTML 中的 `<img>` 使用浏览器默认策略，容易在不同设备/网络下出现不一致的首屏竞争与 CLS 风险。
   - 目标：要求每个 `<img>` 显式声明 `loading` 与 `decoding`，让性能策略“可审计、可回归”。

2. **原子设计缺少落地约定**
   - 现状：项目已有完整的 token/组件样式，但缺少一份把“Atoms/Molecules/Organisms”映射到现有 CSS 体系的明确约定，后续扩展容易产生“私有样式漂移”。
   - 目标：在 `docs/STYLE_GUIDE.md` 中补齐 Atomic Design 分层与实践准则，作为长期可维护的协作规范。

所有变更必须遵循开闭原则：不破坏现有页面结构与业务逻辑，仅通过增量式门禁与规范收敛提升质量与性能上限。
