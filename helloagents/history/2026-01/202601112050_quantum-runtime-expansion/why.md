# 变更提案: 量子运行时扩展与体验重构

## 需求背景
现有版本已具备稳定的静态站点能力，但缺少更具行动指导性的“行为画像 + 决策节奏”层，且视觉系统需要提升整体沉浸感与一致性。为满足高密度信息编排与更强的本地决策支持，需要在保持核心架构不变的前提下，扩展运行时逻辑与前端呈现。

## 产品分析

### 目标用户与场景
- **用户群体:** 需要高效率整理游戏攻略与路线的玩家、内容策展者、深度爱好者
- **使用场景:** 规划通关路线、探索新游戏/攻略、追踪更新、参与社区讨论
- **核心痛点:** 信息分散、决策成本高、缺少节奏与影响力可视化

### 价值主张与成功指标
- **价值主张:** 用“DNA/动量/冲刺/热度/影响力”将行为数据转译为可行动决策
- **成功指标:** 推荐命中率提升、路线规划更易用、更新中心更可读、页面加载更稳健

### 人文关怀
坚持 Local-first，不上传任何用户行为数据，避免隐私风险并尊重离线使用场景。

## 变更内容
1. 新增行为画像与动量节奏（Dashboard / Discover / Guide / Game）
2. 引入意图驱动探索推荐与路线生成策略
3. 规划器加入智能冲刺节奏与排序策略
4. 社区热度雷达与话题画像输出
5. 更新中心增加影响力评分与分级提示
6. 视觉系统升级为 Quantum Glass + Bento

## 影响范围
- **模块:** runtime
- **文件:** `styles.css`, `scripts.js`, `dashboard.html`, `discover.html`, `planner.html`, `community.html`, `game.html`, `guide-detail.html`, `updates.html`, `README.md`, `CHANGELOG.md`
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 行为画像与动量节奏
**模块:** runtime
基于收藏/进度/路线等本地状态生成 DNA 与动量节奏标签。

#### 场景: 玩家进入指挥舱
已存在收藏/进度数据。
- 自动展示画像与动量提示

### 需求: 意图驱动探索推荐
**模块:** runtime
按用户意图标签合并权重，生成探索推荐与路线入口。

#### 场景: 玩家选择“沉浸叙事”意图
已选择意图标签。
- 推荐结果优先展示相关类型游戏与攻略

### 需求: 智能冲刺规划
**模块:** runtime
依据专注时长与内容结构生成冲刺计划。

#### 场景: 玩家在 Planner 生成冲刺节奏
已配置专注时长。
- 自动输出分段路线与复制摘要

### 需求: 社区热度与话题画像
**模块:** runtime
以参与度与更新时间构建热度雷达与画像标签。

#### 场景: 玩家浏览社区页面
页面已加载话题数据。
- 显示热度排名与画像提示

### 需求: 更新影响力评分
**模块:** runtime
综合更新时效与阅读量形成影响力指标。

#### 场景: 玩家查看更新中心
有新增/更新条目。
- 显示影响力等级与进度条

## 风险评估
- **风险:** 复杂评分逻辑导致推荐偏差或性能波动
- **缓解:** 使用本地缓存与轻量计算、增加降级阈值、基础语法检查与页面结构校验
