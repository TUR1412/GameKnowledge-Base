# 变更提案: 贡献指南 + Docs Portal 扩展（离线可读）

## 需求背景
项目当前已经具备稳定的“纯静态产品化”能力（PWA、离线缓存、工具链自检、数据模型校验、交互式文档入口）。随着功能与数据规模扩大，维护者与贡献者需要一条更明确、可复制、可回归的贡献路径：

- **如何添加内容而不破坏数据模型**
- **如何在本地跑完 CI 同款校验**
- **何时必须 bump 版本号以避免缓存幽灵**
- **如何保持不引外链资源的稳定交付约束**

与此同时，站内 `docs.html` 已提供交互式文档入口，但缺少“贡献指南”的一站式入口与离线预缓存支持。

## 变更内容
1. 新增贡献指南文档（`docs/CONTRIBUTING.md`），覆盖内容贡献与工程贡献两类路径。
2. 在 Docs Portal 中新增 `?doc=CONTRIBUTING` 入口，并加入导航列表。
3. 将贡献指南加入 Service Worker 预缓存（带版本号），保证离线可读。
4. README 同步：补齐贡献入口链接与示例版本号，降低新贡献者上手成本。

## 影响范围
- **模块:**
  - 文档系统（README / docs）
  - Docs Portal（docs.html + scripts.js）
  - 离线预缓存（sw.js）
- **文件:**
  - `docs/CONTRIBUTING.md`
  - `CONTRIBUTING.md`
  - `docs.html`
  - `scripts.js`
  - `sw.js`
  - `README.md`

## 核心场景

### 需求: 贡献路径清晰可复制
**模块:** 文档/贡献流程

#### 场景: 新贡献者第一次提交内容
- 预期结果：可以按步骤完成本地安装、校验、版本穿透与提交
- 预期结果：不需要理解全部架构也能安全扩展数据

### 需求: 站内文档入口一站式覆盖
**模块:** Docs Portal

#### 场景: 维护者在站内直接查阅贡献指南
- 预期结果：支持 `docs.html?doc=CONTRIBUTING` 直达
- 预期结果：离线可读（SW 预缓存覆盖）

## 风险评估
- **风险:** 增加新的文档文件导致 SW 预缓存校验失败
  - **缓解:** 更新 `sw.js` 的 `PRECACHE_URLS` 并跑 `npm run check:all` 保障一致性
- **风险:** 更新版本号后 README 示例过期
  - **缓解:** 在 bump 后同步更新 README 中的示例版本号
