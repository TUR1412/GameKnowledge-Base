# 变更提案: UI/UX 视觉革命 v3（Aurora Glass）

## 需求背景
项目已经具备完整的静态站点交付能力（PWA/离线/命令面板/虚拟列表/工具链自检），但当前视觉体系经历了多轮“末尾覆盖式”迭代后，存在以下可感知问题：

1. **品牌与组件风格出现漂移**：存在多套渐变/阴影/高光参数并存，导致不同页面与不同组件的“质感”不完全一致。
2. **关键组件仍有可提升的“世界级细节”**：按钮/卡片/导航等交互密集区的动效手感、边界层级、hover/active 反馈可以进一步统一与精细化。
3. **可访问性与降级策略需要更明确的“SSOT”**：在高对比度、Reduced Motion、低性能设备等场景下，毛玻璃与动态边框需要可控降级，避免额外合成层带来的压力。

本次目标是以“设计系统”为核心，把已有的 glass/bento 方向提升到更统一、更可维护、更“行业标杆”的视觉与交互质量。

## 变更内容
1. **设计系统收敛**：统一渐变边框/光晕/阴影阶梯的 token 作为全站 SSOT，消除“颜色体系漂移”。
2. **关键组件升级**：按钮、Icon Button、Badge/Chip、Hero/Banner、卡片容器等统一为 Aurora Glass 风格（轻量毛玻璃 + 流光边框 + 可控抬升）。
3. **微交互一致性**：统一 hover/active/press 的位移、阴影、过渡曲线与 Reduced Motion 降级策略。
4. **可访问性加固**：确保高对比度模式下关闭动态边框与毛玻璃（更清晰边界与文本），保持可读性优先。

## 影响范围
- **模块:**
  - 样式系统（全站 Design Tokens / 组件样式）
  - 交互视觉一致性（hover/active/press、焦点可见性）
- **文件:**
  - `styles.css`（设计系统 token 与组件样式收敛）
  - （如需）`scripts.js`（仅在必要时补齐与视觉相关的小型交互一致性逻辑）
  - `docs/STYLE_GUIDE.md`（如新增/调整了视觉约定，需同步）

## 核心场景

### 需求: 统一 Aurora Glass 视觉体系
**模块:** UI/样式系统

#### 场景: 用户在不同页面切换浏览
- 预期结果：页面之间“质感一致”（同层级组件阴影/边界/玻璃强度一致）
- 预期结果：主色/辅色/强调色在卡片、按钮、提示等组件中表达一致，不出现“跳色”

### 需求: 微交互世界级手感
**模块:** UI/交互

#### 场景: 用户高频点击按钮、卡片、筛选项
- 预期结果：hover/active/press 的反馈统一且克制（不抖、不飘、不迟钝）
- 预期结果：Reduced Motion 时自动降级（不强制位移/动画），但仍保持清晰反馈

### 需求: 可访问性优先的降级策略
**模块:** UI/可访问性

#### 场景: 高对比度 / 强光 / 强制色彩模式
- 预期结果：关闭毛玻璃与动态边框，边界更清晰、文本对比更强
- 预期结果：焦点可见性强（键盘导航不迷失）

## 风险评估
- **风险:** 视觉 token 收敛可能影响全站大量组件的观感与层级
  - **缓解:** 采用“末尾覆盖”方式渐进收敛；每步改动后跑 `npm run check:all` 做全站一致性校验
- **风险:** 过度使用毛玻璃/动效导致性能回退
  - **缓解:** 以 `--glass-*` token 统一控制；高对比/低动效场景降级；避免不必要的合成层（例如 skeleton card 禁用 blur）
