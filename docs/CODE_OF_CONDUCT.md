# 行为准则 / Code of Conduct

---

## 中文（CN）

### 我们的承诺

我们致力于营造一个友善、包容、专业的协作环境。无论你是新手还是长期贡献者，都应当被尊重对待。

### 适用范围

本准则适用于本仓库的所有协作场景，包括但不限于：Issues、Pull Requests、代码评审、讨论区以及任何与项目相关的交流。

### 鼓励的行为

- 尊重他人，保持建设性沟通
- 聚焦问题本身，提出可执行的建议
- 理性讨论分歧，给出事实与依据
- 帮助新贡献者理解项目规范

### 不可接受的行为

- 任何形式的骚扰、歧视、仇恨言论或人身攻击
- 公开他人隐私信息（doxxing）或威胁他人
- 刷屏、灌水、恶意引战、与项目无关的广告
- 其他维护者认为会破坏社区秩序的行为

### 处置与执行

维护者有权（并有责任）在必要时采取措施，包括但不限于：编辑/删除内容、锁定讨论、拒绝合并、限制或封禁参与。

### 如何报告

如需报告不当行为：

- 请优先通过 GitHub Issues 联系维护者（避免公开敏感信息与个人隐私）。
- 如涉及安全问题，请参阅 `docs/SECURITY.md` 的建议渠道。

## English (EN)

### Our Pledge

We are committed to a friendly, inclusive, and professional collaboration environment. Everyone should be treated with respect.

### Scope

This code of conduct applies to all project spaces and communications, including Issues, Pull Requests, code reviews, and discussions.

### Expected Behavior

- Be respectful and constructive
- Focus on the problem, propose actionable suggestions
- Discuss disagreements with facts and rationale
- Help newcomers learn the project norms

### Unacceptable Behavior

- Harassment, discrimination, hate speech, or personal attacks
- Doxxing or threats
- Spam, trolling, unrelated advertising
- Any other behavior that maintainers deem disruptive

### Enforcement

Maintainers may take appropriate action when needed, including removing content, locking threads, rejecting contributions, or banning participants.

### Reporting

If you need to report a violation:

- Contact maintainers via GitHub Issues (avoid sharing sensitive personal information publicly).
- For security-related matters, see `docs/SECURITY.md`.
